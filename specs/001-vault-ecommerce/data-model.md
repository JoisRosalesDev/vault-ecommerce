# Data Model Design: VAULT E-commerce

## 1. Prisma Schema Specification

```prisma
datasource db {
  provider  = "postgresql"
  url       = env("DATABASE_URL")
  directUrl = env("DIRECT_DATABASE_URL")
}

generator client {
  provider = "prisma-client-js"
}

/// Define user role privileges for RBAC
enum Role {
  ADMIN
  CLIENT
}

/// Status lifecycle for customer purchases
enum OrderStatus {
  PENDING
  PAID
  FAILED
}

/// Maps to Supabase Auth users profiles
model User {
  id        String   @id @db.Uuid // Must match Supabase Auth User ID
  email     String   @unique
  role      Role     @default(CLIENT)
  orders    Order[]
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  @@map("users")
}

/// Product catalog available to clients and managed by admins
model Product {
  id          String      @id @default(uuid()) @db.Uuid
  name        String
  description String
  price       Decimal     @db.Decimal(10, 2)
  stock       Int         @default(0)
  images      String[]    // Array of Supabase Storage URLs
  isActive    Boolean     @default(true)
  orderItems  OrderItem[]
  createdAt   DateTime    @default(now())
  updatedAt   DateTime    @updatedAt

  @@map("products")
}

/// Customer transaction and checkout records
model Order {
  id               String      @id @default(uuid()) @db.Uuid
  userId           String      @db.Uuid
  user             User        @relation(fields: [userId], references: [id], onDelete: Cascade)
  stripeSessionId  String      @unique // Unique Stripe session reference for payment tracking
  totalAmount      Decimal     @db.Decimal(10, 2)
  status           OrderStatus @default(PENDING)
  orderItems       OrderItem[]
  createdAt        DateTime    @default(now())
  updatedAt        DateTime    @updatedAt

  @@map("orders")
}

/// Line items in a specific order, capturing historical prices
model OrderItem {
  id        String   @id @default(uuid()) @db.Uuid
  orderId   String   @db.Uuid
  order     Order    @relation(fields: [orderId], references: [id], onDelete: Cascade)
  productId String   @db.Uuid
  product   Product  @relation(fields: [productId], references: [id], onDelete: Restrict)
  quantity  Int
  priceAtPurchase Decimal @db.Decimal(10, 2) // Snapshotted product price at time of purchase
  createdAt DateTime @default(now())

  @@map("order_items")
}
```

***

## 2. Integrity & Validation Rules

1.  **Historical Pricing Invariant**:
    -   `priceAtPurchase` on `OrderItem` is a read-only field post-creation. It stores a hard snapshot of `Product.price` at the exact second the purchase was finalized. Changing `Product.price` in the catalog later must not impact the totals of existing orders.
2.  **Product Stock Mutex**:
    -   Reducing product stock during purchase must use transactional locks. When an order transitions from `PENDING` to `PAID`, the application must decrement `Product.stock` by the purchased `quantity`. If stock falls below 0, the database transaction must roll back.
3.  **Supabase Auth Sync**:
    -   `User.id` MUST exactly match the `uuid` generated by Supabase Auth upon signup. This sync will be performed by a PostgreSQL trigger or an API hook during the first successful Google Provider authentication middleware intercept.
4.  **Admin Protection**:
    -   Any product mutation (create, update, delete, publish) requires verification that `User.role` is `ADMIN`.

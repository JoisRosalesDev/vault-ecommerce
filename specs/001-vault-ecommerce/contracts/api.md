# API Interface Contracts: VAULT E-commerce

All API communication must be strictly typed, handle timeouts using `AbortController`, and output user-facing messages in Spanish.

## 1. Stripe Checkout Session Initiation

### `POST /api/checkout`
Initiates a secure Stripe Checkout session. Required only for Clients with active shopping carts.

*   **Authentication**: Bearer Token (Supabase JWT) or valid session cookie.
*   **Request Headers**:
    *   `Content-Type: application/json`
*   **Request Payload**:
    ```typescript
    interface CheckoutRequest {
      items: Array<{
        productId: string; // UUID of product
        quantity: number;  // Must be >= 1
      }>;
    }
    ```
*   **Response (HTTP 200 - Success)**:
    ```typescript
    interface CheckoutResponse {
      sessionId: string; // Stripe Session ID
      checkoutUrl: string; // Secure Stripe URL for client redirection
    }
    ```
*   **Response (HTTP 400 - Validation Fail)**:
    ```typescript
    interface CheckoutErrorResponse {
      error: string; // Spanish user-facing message, e.g., "Cantidad no válida"
      code: "INVALID_QUANTITY" | "OUT_OF_STOCK" | "PRODUCT_NOT_FOUND";
    }
    ```
*   **Response (HTTP 429 - Rate Limited)**:
    ```typescript
    interface RateLimitResponse {
      error: "Estamos procesando un alto volumen de solicitudes. Por favor, inténtalo de nuevo en unos segundos.";
    }
    ```

## 2. Stripe Webhook Handler

### `POST /api/webhooks/stripe`
Asynchronous listener receiving Stripe events. Must be public but verified.

*   **Authentication**: Signature verification using `stripe-signature` header.
*   **Request Headers**:
    *   `stripe-signature`: Required cryptographic hash generated by Stripe.
*   **Request Payload**:
    Standard Stripe webhook JSON payload representing a `checkout.session.completed` event.
*   **Response (HTTP 200 - Success)**:
    ```json
    {
      "received": true
    }
    ```
*   **Response (HTTP 400 - Signature Verification Failed)**:
    ```json
    {
      "error": "Firma de webhook no válida"
    }
    ```

## 3. Admin Product Management CRUD

### `POST /api/products` (Create)
Allows authenticated admin users to create a new product.

*   **Authentication**: Admin role session required (validated via middleware edge check).
*   **Request Payload**:
    ```typescript
    interface CreateProductRequest {
      name: string;
      description: string;
      price: number; // Decimal format
      stock: number; // Integer >= 0
      images: string[]; // List of Supabase Storage bucket URLs
    }
    ```

### `PUT /api/products/[id]` (Update)
Allows authenticated admin users to modify an existing product's catalog details.

*   **Request Payload**: Same properties as `CreateProductRequest` but optional.

### `DELETE /api/products/[id]` (Delete)
Performs soft delete or hard delete of a product. If referenced in past orders, must be marked `isActive = false` rather than hard deleted to prevent database violation.
